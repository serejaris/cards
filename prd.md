# PRD: Личная база фактов и генератор flash cards для ЧГК

## 1. Идея

Продукт помогает быстро собирать, хранить и повторять факты перед играми в «Что? Где? Когда?».

Пользователь может задавать темы, добавлять свои материалы, сохранять ошибки после игр и постепенно строить личную базу знаний. На основе этой базы система генерирует классические flash cards: вопрос → ответ → пояснение → повторение.

## 2. Целевая аудитория

Игроки в ЧГК и похожие интеллектуальные игры, которым нужно быстро запоминать разрозненные факты, культурные связи, имена, даты, произведения, смерти, открытия, сюжеты и контексты.

В первую очередь — один пользователь: ты сама.

## 3. Главная задача MVP

Сделать личную базу фактов, где можно:

1. Создать тему.
2. Добавить факты вручную или сгенерировать их по теме.
3. Превратить факты в flash cards.
4. Тренироваться по карточкам.
5. Отмечать карточки как «знала», «не знала», «повторить».
6. Видеть слабые темы и возвращаться к ним.

Текущая реализация MVP — компонентное приложение на React + Vite. Данные MVP хранятся локально в `localStorage`.

## 4. Не делаем в MVP

Пока не делаем:

- сложную геймификацию;
- полноценную социальную систему;
- маркетплейс колод;
- сложный spaced repetition по всем правилам Anki;
- генерацию уникального визуального дизайна для каждой карточки;
- режим ЧГК-вопросов с подвохом;
- мобильное приложение.

## 5. Основные сущности

### Тема

Например:

- католические святые и как они умерли;
- культовые компьютерные игры;
- пьесы Шекспира;
- античные философы;
- художники Возрождения;
- странные смерти исторических персонажей.

Поля:

- название;
- описание;
- теги;
- дата создания;
- количество фактов;
- количество карточек;
- статус изучения.

### Факт

Единица знания внутри темы.

Поля:

- краткий факт;
- подробное объяснение;
- источник, если есть;
- связанные люди / события / произведения;
- теги;
- уровень важности;
- заметка пользователя.

### Карточка

Flash card, созданная из факта.

Поля:

- вопрос;
- ответ;
- пояснение;
- тема;
- связанный факт;
- статус: новая / знала / не знала / повторить;
- количество повторений;
- дата последнего повторения.

### Ошибка

Фиксация слабого места после тренировки или игры.

Поля:

- что не вспомнила;
- правильный ответ;
- тема;
- комментарий;
- дата;
- связанная карточка или факт.

## 6. Пользовательские сценарии

### Сценарий 1: создать колоду по теме

1. Пользователь вводит тему: «Культовые компьютерные игры 90-х».
2. Система предлагает список фактов.
3. Пользователь выбирает, что оставить.
4. Система создаёт карточки.
5. Пользователь запускает тренировку.

### Сценарий 2: добавить свои материалы

1. Пользователь вставляет текст, список фактов или заметку.
2. Система выделяет из материала факты.
3. Пользователь подтверждает или редактирует.
4. Система создаёт карточки.

### Сценарий 3: тренировка

1. Пользователь выбирает тему.
2. Видит вопрос на карточке.
3. Нажимает «показать ответ».
4. Видит ответ и пояснение.
5. Отмечает: «знала», «не знала», «повторить».
6. Система обновляет статус карточки.

### Сценарий 4: работа с ошибками

1. После игры пользователь добавляет: «Не вспомнила святую Агату».
2. Система предлагает найти или создать факт.
3. Создаёт карточку.
4. Помечает её как приоритетную для повторения.

## 7. Основные экраны MVP

Текущий прототип состоит из семи экранов в `src/screens/`. Каждый экран должен быть описан в PRD и доступен из боковой навигации, если это основной раздел приложения.

### Главная (`home`)

Назначение: дать обзор личной базы и быстрый вход в повторение.

Основные блоки:

- боковая навигация;
- заголовок «Мои темы»;
- быстрые действия: «Создать тему», «Быстрая тренировка»;
- блок «Повторить сегодня»;
- блок «Слабые темы»;
- таблица тем.

Компоненты:

- `SidebarNavigation`;
- `ScreenHeader`;
- `QuickActionButton`;
- `ReviewTodayCard`;
- `WeakTopicsCard`;
- `TopicTable`;
- `TopicTableRow`.

Действия и переходы:

- «Быстрая тренировка» открывает `training`;
- «Начать сессию» открывает `training`;
- выбор темы ведёт к `topic`;
- «Создать тему» остаётся будущим действием MVP.

### Экран темы (`topic`)

Назначение: показать одну тему, её факты, карточки и действия для обучения.

Основные блоки:

- ссылка назад к списку тем;
- шапка темы: название, описание, теги, стиль колоды;
- действия: добавить факт, сгенерировать, начать тренировку;
- метрики темы: факты, карточки, процент освоения;
- табы «Факты» и «Карточки»;
- список фактов.

Компоненты:

- `BackLink`;
- `TopicHeader`;
- `TopicTagList`;
- `TopicActionPanel`;
- `MetricCard`;
- `TopicTabs`;
- `FactList`;
- `FactRow`.

Действия и переходы:

- «Назад к списку тем» открывает `home`;
- «Добавить факт» открывает `import`;
- «Сгенерировать» открывает `generate`;
- «Начать тренировку» открывает `training`;
- «Редактировать» остаётся будущим действием MVP.

### Добавить материал (`import`)

Назначение: принять текст, заметку или список фактов для последующего извлечения.

Основные блоки:

- заголовок экрана;
- выбор темы;
- текстовое поле для материала;
- кнопка «Извлечь факты».

Компоненты:

- `ImportForm`;
- `TopicSelect`;
- `MaterialTextarea`;
- `PrimaryActionButton`.

Действия и переходы:

- «Извлечь факты» открывает `generate`;
- выбор «Создать новую тему…» остаётся будущим действием MVP.

### Подготовка фактов (`generate`)

Назначение: показать извлечённые или сгенерированные факты перед сохранением.

Основные блоки:

- заголовок экрана;
- поле с названием темы;
- кнопка «Обновить»;
- список найденных фактов с чекбоксами;
- действия «Сохранить факты» и «Отмена».

Компоненты:

- `GeneratedFactsHeader`;
- `TopicNameInput`;
- `ExtractedFactList`;
- `ExtractedFactCheckboxRow`;
- `SaveFactsButton`;
- `CancelButton`.

Действия и переходы:

- «Сохранить факты» сохраняет выбранные факты в тему;
- «Отмена» открывает `topic`;
- «Обновить» повторно запускает подготовку фактов.

### Тренировка (`training`)

Назначение: провести пользователя через карточки выбранной темы.

Основные блоки:

- верхняя строка прогресса: тема, номер карточки, стиль колоды;
- состояние вопроса;
- состояние ответа;
- кнопки результата;
- нижние действия: пропустить, завершить.

Компоненты:

- `TrainingProgressBar`;
- `TrainingCardFront`;
- `TrainingCardBack`;
- `AnswerRevealButton`;
- `TrainingResultButtons`;
- `TrainingFooterActions`.

Действия и переходы:

- «Показать ответ» переключает карточку в состояние ответа;
- «Знала», «Не знала», «Повторить» фиксируют результат и показывают следующую карточку;
- «Пропустить» переходит к следующей карточке без результата;
- «Завершить» открывает `home`.

### Ошибки (`errors`)

Назначение: хранить слабые места после игр и тренировок.

Основные блоки:

- заголовок «Журнал ошибок»;
- действие «Записать»;
- таблица ошибок: контекст, верный ответ, тема, дата, действие.

Компоненты:

- `ErrorsHeader`;
- `AddErrorButton`;
- `ErrorsTable`;
- `ErrorTableRow`;
- `CreateCardFromErrorButton`.

Действия и переходы:

- «Записать» открывает будущую форму добавления ошибки;
- «Создать карту» создаёт карточку из ошибки или открывает подготовку карточки.

### Статистика (`stats`)

Назначение: показать прогресс и сложные карточки.

Основные блоки:

- заголовок «Аналитика прогресса»;
- сводные метрики: темы, факты, карты, повторить;
- список сложных вопросов.

Компоненты:

- `StatsHeader`;
- `StatsMetricGrid`;
- `StatsMetricTile`;
- `HardQuestionList`;
- `HardQuestionRow`.

Действия и переходы:

- выбор сложного вопроса в будущем открывает карточку или тренировку по слабым местам;
- метрика «Повторить» в будущем ведёт к тренировке по карточкам со статусом «повторить».

## 8. Компонентная структура

### Глобальные компоненты

- `SidebarNavigation` — постоянная боковая навигация по основным экранам.
- `ScreenContainer` — общий контейнер экрана с максимальной шириной и вертикальными отступами.
- `ScreenHeader` — заголовок, подзаголовок и действия текущего экрана.
- `PrimaryActionButton` — главное действие экрана.
- `SecondaryActionButton` — вторичное действие или отмена.
- `MetricCard` / `StatsMetricTile` — компактная карточка числа и подписи.
- `TopicTableRow` — строка темы на главной.
- `FactRow` — строка факта внутри темы.
- `StatusBadge` — бейдж статуса, тега или стиля колоды.

### Компоненты экранов

- `ReviewTodayCard` — блок «Повторить сегодня» на главной.
- `WeakTopicsCard` — блок слабых тем на главной.
- `TopicTabs` — переключатель между фактами и карточками внутри темы.
- `ImportForm` — форма добавления материала.
- `ExtractedFactCheckboxRow` — найденный факт с чекбоксом в подготовке фактов.
- `TrainingCardFront` — передняя сторона карточки с вопросом.
- `TrainingCardBack` — обратная сторона карточки с ответом и пояснением.
- `TrainingResultButtons` — кнопки «Знала», «Не знала», «Повторить».
- `ErrorTableRow` — строка журнала ошибок.
- `HardQuestionRow` — строка сложного вопроса в статистике.

### Правило синхронизации

Если в `src/screens/` появляется новый экран, он должен быть добавлен в раздел 7 этого PRD. Если на экране появляется новый повторяемый UI-блок, он должен быть добавлен в раздел 8 и вынесен в `src/components/`, если используется больше одного раза.

## 9. Логика генерации карточек

Из каждого факта система должна создавать карточку в формате:

- вопрос;
- ответ;
- пояснение;
- теги.

Пример:

Факт: Святой Лаврентий, по преданию, был казнён на раскалённой решётке.

Карточка:

- Вопрос: Какой святой, по преданию, был казнён на раскалённой решётке?
- Ответ: Святой Лаврентий.
- Пояснение: Его мученичество часто изображается в искусстве через атрибут решётки, поэтому этот образ может быть ключом в вопросах.

## 10. Визуальный стиль

В MVP визуальный стиль должен быть простым, но с заделом на будущие тематические колоды.

Базовые стили колод:

1. Classic Study — чистые карточки для повторения.
2. Old Paper — коричневая бумага, книжный стиль, подходит для Шекспира, святых, истории.
3. Pixel / Games — пиксельная эстетика для компьютерных игр и поп-культуры.

В MVP можно реализовать выбор темы оформления на уровне колоды, а не генерировать уникальный дизайн каждой карточки.

## 11. Приоритеты MVP

### P0 — обязательно

- создание темы;
- добавление фактов;
- генерация карточек;
- тренировка по карточкам;
- статусы «знала / не знала / повторить»;
- хранение данных локально или в простой базе;
- список тем и карточек.

### P1 — желательно

- импорт текста;
- AI-извлечение фактов;
- AI-генерация карточек;
- экран ошибок;
- простая статистика;
- фильтр по слабым карточкам.

### P2 — позже

- красивые тематические колоды;
- spaced repetition;
- генерация ЧГК-вопросов;
- экспорт в Anki;
- импорт PDF / ссылок;
- мобильная версия;
- публичные колоды.

## 12. Критерий готовности MVP

MVP готов, если можно за один вечер:

1. Создать тему «Культовые компьютерные игры».
2. Добавить 20 фактов.
3. Сгенерировать из них 20 карточек.
4. Пройти тренировку.
5. Отметить часть карточек как «не знала».
6. Потом открыть список слабых карточек и повторить их.
